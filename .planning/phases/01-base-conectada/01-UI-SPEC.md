## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Gaps internos de badge, icon gap em `.nav-item` |
| sm | 8px | Padding compacto, gap entre botões, margem de labels |
| md | 16px | Padding padrão de cards (`.num-card`), padding lateral de items da lista |
| lg | 24px | Padding do `.login-box`, seções internas de view |
| xl | 32px | Separação entre seções da home (`.home-date margin-bottom`) |
| 2xl | 48px | Margem do logo no login (`.login-logo margin-bottom`) |
| 3xl | 64px | Não usado nesta fase |

Exceptions:
- Touch targets mínimos: 44px altura para `.nav-item` e `.btn-primary` (acessibilidade mobile)
- Sidebar largura: 200px colapsada para 0 com `transform: translateX(-200px)` — valor fixo, não é token de espaçamento

**Fonte:** extraído diretamente do CSS existente em `index.html`.

## Interaction Contracts

### Navegação SPA

- Função: `irPara(view)` — não substituir
- Efeito: remove `.active` de todas as views, adiciona `.active` na view alvo
- CSS: `.view { display: none }` / `.view.active { display: block }`
- Sem transição de fade — troca instantânea (padrão já adotado)

### Login

- Submit via Enter ou clique no botão
- Estado de loading: opacidade do botão reduzida + texto "aguarde..."
- Erro: exibir em `.login-erro` (já existe no HTML)
- Auto-login: verificar `localStorage.dms_token` no IIFE no final do script — preservar

### Sidebar

- Colapso via `toggleSidebar()` — estado salvo em `localStorage['dms-sb-collapsed']`
- Mobile: sidebar desliza com `transform: translateX(-200px)` — overlay fecha ao clicar fora
- Item ativo: `border-left: 2px solid rgba(255,255,255,0.6)` + `background: rgba(255,255,255,0.05)`

### Verificação de conexão (CORE-03)

- Executar `verificarConexoes()` dentro de `iniciarApp()` após login bem-sucedido
- Output apenas no `console.log` — nenhum indicador visual nesta fase
- Usar `Promise.allSettled` — falha de um não bloqueia os outros
- Verificar `response.ok` (HTTP 200), não `array.length` (RLS pode retornar `[]` com status 200)

### PWA

- Service Worker: `Network-first com fallback` — não alterar estratégia
- Cache busting: `deploy.sh` substitui `CACHE_NAME` via Python (não sed — gotcha Windows)
- Manifest: adicionar PNG 192x192 e 512x512 + `<link rel="apple-touch-icon">` no head
- Nome da app: "DM.Stack" (manifest `short_name`)

## Mobile-First Contract

Esta é uma PWA de uso solo (Duam), instalada no celular como app principal:

- Breakpoint primário: 390px (iPhone 14)
- Sidebar: overlay deslizante no mobile, colapsa completamente
- Touch targets mínimos: 44px de altura para todos os elementos interativos
- Content area: `margin-left: 200px` no desktop, `margin-left: 0` no mobile
- Viewport: `<meta name="viewport" content="width=device-width, initial-scale=1">`
- Instalação PWA: prompt disponível no Android (Chrome) e iOS (via apple-touch-icon)
