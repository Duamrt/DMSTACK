Reescrever deploy.sh para incluir cache busting via Node.js (decisao do usuario — NAO usar Python):

```bash
#!/bin/bash
MSG="${1:-deploy}"
cd "$(dirname "$0")"

# Cache busting: atualiza CACHE_NAME no sw.js
VER="dmstack-$(date +%m%d%H%M)"
node -e "
const fs = require('fs');
let sw = fs.readFileSync('sw.js', 'utf8');
sw = sw.replace(/const CACHE='[^']+'/, \"const CACHE='\" + process.argv[1] + \"'\");
fs.writeFileSync('sw.js', sw);
" "$VER"

git add -A
git commit -m "$MSG"
git push origin main
echo "no ar — $VER"
```

Pontos criticos:
- Usar `node -e` com `process.argv[1]` para receber a versao (seguro, sem problemas de encoding)
- O regex substitui qualquer valor entre aspas simples apos `const CACHE=`
- Formato da versao: `dmstack-MMDDHHMM` (ex: dmstack-04101430)
- Manter `echo "no ar"` no final (padrao do Duam)
- NAO usar sed (gotcha Windows com emojis/encoding)
- NAO usar python3 (decisao do usuario: Node.js)
